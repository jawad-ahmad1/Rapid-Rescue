Rapid Rescue Navigation Test

This page allows you to directly test the navigation page with sample data. If the normal workflow isn't working, you can use these direct links to test different parts of the navigation system.

Test Options

  1. First, start the development server by running npm run dev in your terminal from the Rapid-Rescue directory.
  2. Try the sample data test, which provides test data directly to the navigation component: Navigation Test with Sample Data
  3. If that works, try the main navigation page which uses fallback data if no alert is provided: Main Navigation Page
  4. Compare with the normal flow by starting from the dashboard: Ambulance Dashboard

Troubleshooting

If you see a blank screen:

  1. Open your browser's developer tools (F12 or right-click → Inspect)
  2. Check the Console tab for any error messages
  3. Look for issues related to React, Google Maps API, or script loading

Common Google Maps Issues:

If you see errors about Google Maps, it could be due to:

  • API key issues (invalid or restricted key)
  • Network connectivity problems
  • Script loading failures

The error handling added to the navigation component should display fallback content in these cases.

Development Server Issues:

If you can't start the development server with npm run dev, make sure:

  • You're in the correct directory (Rapid-Rescue)
  • All dependencies are installed (npm install)
  • The dev script is defined in package.json
  • No other process is using port 5173
